## 8. 채택된 핵심 설계 결정 8선

1. **Fallback-first**: PoC·CI는 외부 키·DB 없이 결정론적으로 재현되어야 한다.
2. **하위 호환 우선 (Breaking change 최소화)**: SSE v1 페이로드 유지 + v2 type 이벤트 병행.
3. **관심사 분리**: `app/llm`, `app/retrieval`, `app/security`, `app/memory`, `app/observability` 패키지화.
4. **이중 방어**: API 레이어 1차 차단 + planner 레이어 `_classify_intent` 2차 검증으로 prompt injection·OOS·PII 균형.
5. **운영 관측 가능성**: 구조화 로그, `/metrics` Prometheus 텍스트, HTTP 지연·에러 카운터, 회로 차단 카운터.
6. **의존성 0 옵션**: JWT·CircuitBreaker·in-memory metrics·RAGAS-like를 표준 라이브러리만으로 구현.
7. **LLM 호출 일원화**: 모든 LLM 호출은 `LLMRouter.invoke(intent=...)` 단일 진입점 통과.
8. **evidence 의무 부착**: 모든 구조화 출력에 `evidence_ids: list[str]` 필수, 미부착 응답 0건.
