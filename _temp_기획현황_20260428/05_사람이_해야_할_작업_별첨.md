# 05. 사람이 해야 할 작업 별첨 (Human-Required Tasks)

> **목적**: AI(Claude·Codex 등)가 코드·문서·테스트는 대신 생성할 수 있으나, **접근 권한·정책 결정·외부 계약·운영 승인** 이 필요한 항목은 사람이 직접 수행해야 한다. 본 문서는 그 항목을 카테고리별로 정리하고, 각 항목의 **차단 이유·선결 조건·예상 소요 시간**을 명시한다.
>
> **기준일**: 2026-04-29 (Sprint 7 골격 선도입 이후 기준)
>
> **완료 현황** (2026-04-29 업데이트): 카테고리 1 항목 1-1~1-4 ✅ 완료 / 카테고리 2 항목 2-1~2-2 ✅ 완료 — DART API·LLM·Neo4j·Qdrant 실연동 연결 확인됨.

## 참고: MAX_REFLEXIONS 제한 해제 시 루프 횟수 분석

> **관련 코드**: `app/agents/edges.py`, `app/agents/nodes.py`, `app/agents/graph.py`

### 현재 설계

```
MAX_TURNS     = int(os.getenv("MAX_TURNS", "3"))      # orchestrator 호출 상한
MAX_REFLEXIONS = int(os.getenv("MAX_REFLEXIONS", "2")) # reflector 루프 상한
```

**reflexion 루프 경로** (critic이 재검색 요청 시):

```
retrieve_context → finance_analyst → risk_compliance → critic
        ↑                                                    │ score ≥ 0.5 AND
        └──────────── reflector ─────────────────────────────┘ not has_evidence
```

**루프 조건** (`critic_node`):
```python
request_re_retrieval = bool(
    score >= 0.5                              # 불일치 점수 임계
    and state.get("reflexion_count", 0) < MAX_REFLEXIONS  # ← 이 가드가 핵심
    and not has_evidence                      # fallback 모드에서는 항상 True
)
```

### 기본값(MAX_REFLEXIONS=2) 기준 흐름

| 단계 | reflexion_count | next_node |
|------|-----------------|-----------|
| 최초 critic 판정 | 0 | reflector (루프 진입) |
| reflector 1회 | 1 | retrieve_context |
| critic 재판정 | 1 | reflector (루프 진입) |
| reflector 2회 | 2 | retrieve_context |
| critic 재판정 | 2 | **orchestrator** (count ≥ MAX_REFLEXIONS → 강제 탈출) |

→ **최대 2회 reflexion 루프**, 총 노드 실행 = 5(초기) + 5×2(루프) + 2(orchestrator+report) = **17 steps**

### MAX_REFLEXIONS 제한을 완전히 제거하면?

#### Case A: LocalFallbackGraph (fallback 모드)

`LocalFallbackGraph.astream()`의 안전 가드:
```python
max_steps = max(8, (MAX_TURNS + MAX_REFLEXIONS) * 4)  # 기본값 = 20
```

MAX_REFLEXIONS 코드 체크를 제거해도 **`max_steps`가 두 번째 방어선**으로 작동:

| MAX_REFLEXIONS 환경변수 | max_steps 계산값 | 이론적 최대 루프 수 |
|------------------------|-----------------|-------------------|
| 2 (기본값) | 20 | 2회 |
| 0 | 12 | 1회 (max_steps에 의해 종료) |
| 99 | 408 | ~80회 (단, steps 계산 기준) |
| **코드 체크 제거 + ENV=2** | 20 | **3회** (max_steps가 17→20에서 잡음) |

> **결론**: `reflector_node`의 `count > MAX_REFLEXIONS` 체크를 제거하더라도 `LocalFallbackGraph`는 `max_steps = 20`에서 강제 종료 → **최대 3회** reflexion 루프 허용 후 `generate_final_report` 강제 호출.

#### Case B: LangGraph 모드 (실 LangGraph 설치 시)

LangGraph는 별도 안전 가드로 **`recursion_limit`** 을 가진다 (기본값 = **25 steps**):

```
초기 경로 (intent→retrieve→finance→risk→critic): 5 steps 소모
각 루프 (reflector→retrieve→finance→risk→critic): 5 steps 소모
종료 (orchestrator→generate): 2 steps 소모
```

`5 + 5N + 2 ≤ 25` → `N ≤ 3.6` → **최대 3회 루프** 후 `RecursionError` 발생 (미처리 시 500 에러)

> **결론**: MAX_REFLEXIONS 체크 없이 LangGraph를 사용하면 4번째 루프 시작 시점에 `GraphRecursionError`가 터지며 요청이 실패한다.

### 권장 사항

1. **`reflector_node`의 `count > MAX_REFLEXIONS` 체크는 반드시 유지** — 이것이 1차 의미 있는 탈출 조건.
2. `MAX_REFLEXIONS` 값을 높이면 LangGraph `recursion_limit`과 충돌할 수 있으므로 `MAX_REFLEXIONS × 5 + 7 < recursion_limit` 조건을 만족하도록 설정.
3. LangGraph 사용 시 `graph.invoke(config={"recursion_limit": N})` 으로 명시적 상한 설정 권장.
4. `LocalFallbackGraph`의 `max_steps` 수식 `(MAX_TURNS + MAX_REFLEXIONS) * 4`는 ENV 값에 따라 자동 조정되므로 ENV만 바꿔도 안전 가드가 함께 늘어남에 주의.
