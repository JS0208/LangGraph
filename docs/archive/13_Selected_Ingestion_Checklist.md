# 13. Selected Ingestion Checklist

## 목적

이 문서는 5개 핵심 기업에 대해 `2023~2026` 데이터를 전량 수집하지 않고, 질의 응답 품질에 직접 도움이 되는 정보만 선별 적재하기 위한 기준이다.

## 대상 기업

- 삼성전자 (`005930`)
- SK하이닉스 (`000660`)
- NAVER (`035420`)
- 카카오 (`035720`)
- LG CNS (`064400`)

## 대상 연도

- `2023`
- `2024`
- `2025`
- `2026` (현재 시점까지 공개된 자료만)

## 선별 적재 기준

### 1. 재무 정보

- 연도별 핵심 재무 1건 이상 적재
- 필수 항목:
  - 매출
  - 영업이익
  - 부채비율
- 과거 연도는 `FY` 기준으로 저장
- 진행 중인 연도는 `YTD` 기준으로 저장

### 2. 공시 정보

- 연도별 중요 공시만 선별 적재
- 기본 상한:
  - 기업당 연도별 최대 5건
- 우선 키워드:
  - 규제
  - 소송
  - 계약
  - 투자
  - 증자 / 감자
  - 전환사채 / 신주
  - 인수 / 합병 / 분할
  - 자회사
  - 영업양수도
  - 횡령 / 배임
- 우선 키워드가 없으면 최신 공시에서 상위 몇 건만 유지

### 3. 벡터 적재 기준

- 원문 전체 대신 요약 텍스트 중심 적재
- `company_name`, `year` payload 필수
- 임베딩 차원은 컬렉션 설정과 일치해야 함

### 4. 그래프 적재 기준

- `Company`
- `FinancialReport`
- `Disclosure`
- 관계:
  - `HAS_REPORT`
  - `INVOLVED_IN`

## 운영 체크리스트

- [ ] 5개 회사 종목코드가 정확한가
- [ ] 2023~2026 루프가 설정되어 있는가
- [ ] 현재 연도는 미래 날짜를 조회하지 않는가
- [ ] 연도별 공시 상한이 걸려 있는가
- [ ] 중요 공시 키워드 우선순위가 적용되는가
- [ ] Qdrant payload index(`company_name`, `year`)가 존재하는가
- [ ] Qdrant 임베딩 차원이 컬렉션과 일치하는가
- [ ] Neo4j 적재가 연도별로 중복 없이 `MERGE` 되는가
- [ ] 실패 시 전체 중단이 아니라 가능한 범위까지 적재되는가

## 권장 운영 방식

1. 먼저 5개 회사 `2023~2025`를 적재
2. 이후 `2026` 현재 시점 데이터 추가
3. 실제 질의 로그를 보고 부족한 회사/연도/공시만 증분 확장
