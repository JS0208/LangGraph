## A · 약어 (영문)

| 용어 | 뜻 |
|------|-----|
| **a11y** | **Accessibility**(접근성)의 관례적 표기. 장애인·고령자 등이 동등하게 UI를 쓰도록 하는 설계·검증(예: WCAG, 스크린 리더, 키보드 조작). |
| **API** | **Application Programming Interface**. 백엔드가 클라이언트·다른 시스템과 약속한 호출 규격(엔드포인트, 요청/응답 형식). |
| **As-Is** | **현재 상태**. 업그레이드 전의 역량·한계를 기술할 때 쓰는 기획 용어. |
| **To-Be** | **목표 상태**(미래). As-Is와 대비해 성숙·도입 후 모습을 적을 때 쓰는 기획 용어. |
| **BGE** | **BAAI General Embedding** 계열. 중국 Beijing Academy of AI 등에서 공개한 임베딩·리랭커 모델 패밀리. 문맥에서는 한국어·다국어 리랭킹에 쓰는 **BGE-reranker**·**BGE-m3** 등을 가리킨다. |
| **BM25** | **Best Matching 25**. 전통적인 **키워드(희소)** 검색 점수 함수. dense(벡터) 검색과 **Hybrid**로 섞으면 고유명사·숫자 회수에 유리하다. |
| **CD** | **Continuous Delivery / Deployment**. 빌드·배포를 자동화해 스테이징/프로덕션에 안전하게 옮기는 관행. |
| **CI** | **Continuous Integration**. 코드 머지 시 자동으로 lint·테스트·(확장 시) 골든셋 평가를 돌려 품질을 막는 흐름. |
| **CORS** | **Cross-Origin Resource Sharing**. 브라우저가 다른 도메인의 API를 호출할 때 허용 출처를 제한하는 HTTP 정책. 운영에서는 `*` 대신 **화이트리스트**가 권장된다. |
| **CVE** | **Common Vulnerabilities and Exposures**. 공개된 보안 취약점 ID. **high CVE 0건**은 치명적 알려진 취약점이 이미지/의존성에 없도록 하자는 목표. |
| **Cypher** | **Neo4j** 등에서 쓰는 **그래프 쿼리 언어**. `MATCH`·`RETURN` 등으로 노드·관계를 조회한다. |
| **DAG** | **Directed Acyclic Graph**. 작업 간 의존이 있는 **비순환** 그래프. 데이터 적재 파이프라인을 단계(노드)+화살표(의존)로 표현할 때 쓴다. |
| **DART** | **Data Analysis, Retrieval and Transfer System**. 금융감독원 전자공시 **다트**. 국내 상장사 공시 원문·메타데이터의 대표적 출처. |
| **DML** | **Data Manipulation Language**. DB에서 **삽입·수정·삭제** 등 데이터를 바꾸는 SQL/Cypher 구문 범주. 그래프 질의에서 DML을 막고 **읽기 전용**만 허용하면 사고 면적을 줄인다. |
| **DoD** | **Definition of Done**. “이 일이 끝났다”고 말하기 위한 **합의된 조건 목록**(기획·스프린트 종료 기준). |
| **DPA** | **Data Processing Agreement** (처리위탁·데이터 처리 계약). EU GDPR 등 맥락에서 **개인정보·민감 데이터**를 클라우드·SaaS에 넘길 때 법적 역할과 책임을 정하는 문서. 17번에서는 **OTel·LangSmith** 같은 외부로 트레이스/로그가 나갈 때 검토 대상으로 언급. |
| **DSS** | **Decision Support System**. 숫자·근거·시나리오를 바탕으로 **의사결정을 돕는** 시스템. 단순 챗봇이 아니라 출처·재현성이 중요한 방향. |
| **E2E** | **End-to-End**. 사용자 행동 시작부터 최종 응답까지 **전 구간** 지연·성공률을 말할 때 쓴다. |
| **GDPR** | **General Data Protection Regulation**. EU **일반개인정보보호법**. DPA·PII·데이터 주체 권리 논의와 직결된다. |
| **HITL** | **Human-in-the-loop**. 모델·에이전트 흐름 중간에 **사람이 승인·수정·중단**할 수 있게 설계하는 방식. |
| **HTTP / HTTPS** | 웹 통신 프로토콜. API·브라우저 연동의 기반. |
| **i18n** | **Internationalization**(국제화). 다국어·로캘·문자 방향 등 **여러 언어/지역**을 수용하는 제품 구조. |
| **IdP** | **Identity Provider**. **OAuth2/OIDC** 흐름에서 사용자 인증을 맡기는 외부 주체(예: 사내 SSO, Auth0, Keycloak). |
| **JWT** | **JSON Web Token**. 서명된 JSON 주장(claims)을 HTTP **Bearer** 등으로 넘겨 **무상태(stateless) 인증**을 하는 토큰 형식. |
| **KPI** | **Key Performance Indicator**. 조직이 중요하게 보는 **핵심 성과 지표**. |
| **LLM** | **Large Language Model**. 대규모 언어 모델. 질의응답·요약·JSON 생성 등에 쓰인다. |
| **LLMRouter** | 코드베이스에서 LLM 호출을 **한 곳으로** 모으는 **라우터** 클래스·진입점. 17번 Sprint 1 Exit의 `LLMRouter.invoke` 단일화와 연결. |
| **RBAC** | **Role-Based Access Control**. **역할** 단위로 API·리소스 접근을 나누는 모델. OAuth/IdP·테넌트 설계와 함께 논의된다. |
| **NFR** | **Non-Functional Requirements**. 기능 목록이 아니라 **성능·보안·가용성·규제·운영** 같은 비기능 요구사항. |
| **OAuth2** | **OAuth 2.0**. 제3자 앱이 **리소스 소유자 동의** 하에 API 접근 권한을 받는 인가 프레임워크. **JWT**와 자주 같이 쓰이지만 같은 개념은 아님. |
| **OIDC** | **OpenID Connect**. OAuth2 위에 **신원(identity)** 정보를 표준 클레임으로 실어 주는 레이어(간단히 “로그인 표준”). 17번에 직접 없으나 IdP 논의와 연결된다. |
| **OpenTelemetry** | **관측성**을 위한 표준. **OTel**로 줄여 부름. 트레이스·메트릭·로그를 vendor 중립으로 수집해 Jaeger·Tempo 등으로 보낼 수 있다. |
| **OTel** | **OpenTelemetry** 약칭. |
| **PII** | **Personally Identifiable Information**. 개인을 특정할 수 있는 정보(이름·주민번호·계좌·이메일 등). 마스킹·접근 통제·DPA 대상. |
| **PM** | **Product Manager**. 우선순위·스코프·이해관계자 커뮤니케이션을 담당하는 역할. |
| **Postgres / PostgreSQL** | 오픈소스 **관계형 DB**. LangGraph **checkpointer**·감사 로그 등에 쓰일 수 있다. |
| **PostgresSaver** | **LangGraph**가 제공하는 형태로, 그래프 실행 상태를 **PostgreSQL**에 저장해 재개·감사에 쓰려는 컴포넌트(명칭은 프레임워크 문서 기준). |
| **PR** | **Pull Request**. 코드 머지 전 검토·CI를 거치는 작업 단위(GitHub 등). |
| **p95** | **95번째 백분위수**. 지연 분포에서 **상위 95% 요청**이 이 시간 안에 끝난다는 의미(느린 꼬리 제외한 “대부분” 체감에 가깝게 쓸 때 많음). |
| **pytest** | Python **단위·통합 테스트** 프레임워크. |
| **RAG** | **Retrieval-Augmented Generation**. 검색된 문서를 근거로 LLM이 답을 **근거 있게** 생성하는 패러다임. |
| **RAGAS** | **RAG Assessment** 프레임워크. faithfulness·answer relevancy 등 **RAG 품질 지표**를 자동 산출하는 오픈소스 도구(이름이 고유명사). |
| **React Flow** | React용 **노드·엣지 다이어그램** 라이브러리. 지식 그래프 라이브 뷰 후보 중 하나. |
| **RFC** | **Request for Comments**. 본문에서는 API/SSE 스키마를 **제안·합의 중인 규격**으로 올릴 때 관례적으로 쓴다(IETF RFC와 동일하지 않을 수 있음). |
| **ROI** | **Return on Investment**. 투입 대비 효과. |
| **RO (계정)** | **Read-Only** DB 계정. LLM 생성 쿼리가 있어도 **쓰기 권한**이 없으면 피해를 제한한다. |
| **SLA** | **Service Level Agreement**. 서비스 제공자와 고객 간 **가용성·지연** 등 계약. 17번에서는 **SLO**와 함께 운영 목표 맥락. |
| **SLO** | **Service Level Objective**. 내부적으로 맞추려는 **목표 수치**(예: p95 지연 upper bound). |
| **Snyk** | **취약점 스캔**·의존성 보안 SaaS. |
| **SOTA** | **State of the Art**. 해당 시점 기술의 **최첨단**을 말하는 비공식 표현. 기획에서는 **측정 가능한 NFR**으로 풀어 쓰라고 조언함. |
| **SSE** | **Server-Sent Events**. 서버가 클라이언트로 **단방향 이벤트 스트림**을 보내는 HTTP 표준. 실시간 진행 상황 표시에 흔히 쓴다. |
| **TTFU** | **Time To First Update**(또는 First Useful). 사용자가 질문 후 **첫 의미 있는 피드백**(첫 토큰·첫 노드 시작 등)을 보기까지 시간. |
| **Trivy** | 컨테이너·파일시스템 **CVE 스캔** 오픈소스. |
| **UX** | **User Experience**. 사용 경험 전반(정보 구조·피드백·신뢰·개입 가능성). |
| **W3C** | **World Wide Web Consortium**. 웹 표준(HTML, WCAG 등)을 다루는 국제 컨소시엄. |
| **WCAG** | **Web Content Accessibility Guidelines**. W3C 권고 **웹 접근성 가이드**. 등급·검사 항목이 제품 요구로 들어간다. |

## E · 검색·에이전트·아키텍처 개념

| 용어 | 뜻 |
|------|-----|
| **Critic** | 두 분석 출력을 **비판·검증**하는 에이전트 역할. 불일치·인용 누락 등을 점수화. |
| **dense (검색)** | 텍스트를 **벡터(임베딩)** 로 바꿔 유사도 검색하는 방식. |
| **disagreement_score** | 비평 결과 **의견 불일치 정도**를 0~1 등으로 수치화한 것. 임계 이하면 합의로 볼지 결정. |
| **drill-down** | 요약·상위 뷰에서 **세부 데이터로 파고들기**. |
| **evidence / evidence_id** | 답의 **근거가 된 청크·노드**와 그 **식별자**. citation 추적의 최소 단위. |
| **fallback** | 외부 실패 시 **대체 경로**(캐시·시드·mock)로 응답하는 전략. |
| **Hybrid (검색)** | **dense + sparse(BM25 등)** 를 섞는 검색. α 가중으로 비율 조절(기획안). |
| **Intent (의도 분류)** | 질의가 실적·관계·트렌드·리스크·**범위 외** 등 어디에 해당하는지 분류. |
| **jailbreak** | 안전 가드를 **우회**하려는 프롬프트 공격. 골든셋·adversarial에서 별도 측정. |
| **MAX_REFLEXIONS** | 반성·재계획 루프 **최대 횟수** 상한. 무한 루프 방지. |
| **MAX_TURNS** | 에이전트 **라운드** 상한. 비용·시간 폭주 방지. |
| **multi-hop** | 한 번의 검색이 아니라 **여러 단계**로 그래프·문서를 옮겨 가며 추론하는 질의 유형. |
| **Orchestrator** | **여러** 노드 출력을 **종합**해 합의문·최종 보고 방향을 잡는 역할. |
| **Planner / Query Planner** | 질의를 **sub-query** 등으로 쪼개 검색·도구 계획을 세우는 역할. |
| **prompt injection** | 사용자 입력에 **시스템 지시를 덮어쓰라**는 악의적 텍스트를 넣는 공격. |
| **Reflector** | 비평 결과를 바탕으로 **재검색·재계획**을 일으키는 역할. |
| **rerank / Reranker** | 후보 문서를 **재순위화**해 상위 K를 정밀하게 만드는 모델·서비스. |
| **Retriever** | 검색 서브시스템을 호출해 **evidence**를 가져오는 역할. |
| **sparse (검색)** | 키워드 매칭 중심 **희소** 표현 검색. BM25 등. |
| **thread_id** | 한 번의 대화·분석 세션을 식별하는 **ID**. 재개·감사 상관 키. |
| **공시** | 기업이 규제기관·거래소에 **공개하는 정보**(사업·재무·지배구조 등). DART에 올라오는 자료가 대표적. |
| **시나리오 A / B** | 마스터 플랜의 **골든 시나리오** 예: A는 자회사 리스크 전이, B는 상충 데이터로 **합의 실패**를 명시. |
| **체크포인터 / checkpointer** | 그래프 **상태를 저장**해 재시작·재현·재개에 쓰는 메커니즘. |
| **토큰 (스트리밍)** | LLM 출력의 **부분 문자열** 또는 SSE `token` 이벤트로 쪼갠 조각. 체감 속도에 영향. |
| **3-pane** | 화면을 **세 패널**(예: 질의+로그 / 결론 / 그래프)로 나누는 UI 레이아웃. |
| **Debate timeline** | 에이전트·Critic 등의 **합의·이견 과정**을 시간순으로 보여 주는 UI(기획명). |
| **Evidence Drawer** | 근거·출처 목록을 **서랍/사이드 패널**로 열어 보는 UX 패턴. |
| **lineage** | **데이터 계보**. 소스·파이프라인·버전을 추적하는 메타. Pillar 5 “lineage 메타”와 연결. |
| **Model Router** | 작업 유형에 따라 **모델·프로바이더**를 골라 호출하는 라우팅(추론 vs 추출 등). **LLMRouter**와 같은 계열 개념. |
| **quality gate** | 적재·인덱싱 파이프라인에서 **품질 기준 미달 시 다음 단계를 막는** 검문. |
| **Resume API** | 중단된 그래프 실행을 **같은 thread**로 이어 실행하는 HTTP API(제품 스토리). |
| **alias 스왑** | 검색/벡터 인덱스의 **읽기 별칭(alias)** 을 새 스냅샷으로 **원자적으로** 바꿔 무중단 전환하는 패턴. |
| **chaos (테스트)** | 네트워크·프로세스 등에 **장애를 주입**해 회복·폴백을 검증하는 운영 관행. |
| **rate limit** | API **요청 속도 상한**. 남용·DoS·비용 폭주 완화. |

## H · 지표 이름

| 용어 | 뜻 |
|------|-----|
| **answer_correctness** | **RAGAS** 등에서 정답과의 **정답 정합성**에 가까운 지표. |
| **Cache hit (ratio)** | 캐시에 **적중**해 외부 호출을 줄인 비율. 비용·지연에 직결. |
| **Citation 부착률** | 결론에 **출처 ID**가 붙은 비율. 100%가 목표로 제시됨. |
| **Error rate** | 요청 대비 **오류** 비율. |
| **faithfulness** | 생성 답이 **제공된 근거에 충실한가**를 나타내는 지표(RAGAS 등). 0.85 목표 예시. |
| **Fallback rate** | 전체 요청 중 **폴백 경로**로 간 비율. 높으면 실연동 품질·키 문제 신호. |
| **recall@5** | 상위 5개 후보 안에 **정답 근거**가 들어올 확률(검색 회수율). multi-hop 5건에서 ≥0.9 예시. |
