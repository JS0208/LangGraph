# 추후 고려 사항

## 🔴 높은 우선순위

### 1. Interrupt 후 재개
- 그래프 실행 중 interrupt 발생 시 상태를 보존하고 재개하는 로직 구현 필요
- 신뢰성과 사용자 경험에 직접 영향

### 2. 반대의미 문제
- 유사도 기반 검색에서 반대 의미 단어가 높은 점수를 받는 오류 처리
- 검색 정확도 및 결과 품질에 직접 영향

### 3. 하드 코딩된 프롬프트 텍스트
- 코드 내 하드 코딩된 프롬프트 텍스트 외부화 (config 파일, 템플릿 등)
- 유지보수성과 유연한 프롬프트 관리에 필수

### 4. Guardrails 노드
- 유해하거나 범위를 벗어난 입력에 대한 guardrails 노드 추가 여부 검토
- 시스템 안전성 확보

---

## 🟡 중간 우선순위

### 5. 2-hop으로 불충분한 경우 처리
- 현재 2-hop 탐색이 충분하지 않은 케이스 분석 및 동적 hop 조정 방안 검토

### 6. Sparse Score 통합
- BM25 등 sparse retrieval 점수를 dense 벡터 점수와 결합하는 hybrid 방식 도입 검토

### 7. Reflector Weight 반영
- reflector(반성 모듈)의 피드백 가중치를 그래프 탐색 또는 랭킹에 반영하는 방안 구현

### 8. 노드 구조 재검토
- 노드 분리 vs. 병합 트레이드오프 분석
- 평가(evaluation) 노드 도입 여부 및 전체 노드 수 최적화

---

## 🟢 낮은 우선순위

### 9. 데이터 더 많이 적재
- 현재보다 더 많은 데이터를 벡터/그래프 DB에 적재하여 커버리지 향상

### 10. 관리자용 벡터/그래프 저장 정보 제공
- 관리자가 현재 저장된 벡터 및 그래프 상태를 확인할 수 있는 정보 UI 또는 API 제공

### 11. 미장 (전반적 개선)
- UX, 코드 정리, 출력 형식 등 전반적인 품질 다듬기
