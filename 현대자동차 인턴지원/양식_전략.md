# 자소서 양식 전략 (업데이트: 2026-05-05)
> 제출 형식: PDF, A4 1장, 5MB 이내 | 자유 양식

---

## 0. 서치로 확인한 사실 (양식 선택 근거)

- **채용 담당자는 모바일로 검토하는 비중이 높다** — 가로 2단 레이아웃은
  모바일에서 흐름이 끊겨 가독성 저하. **세로형(단단) 레이아웃이 안전.**
- **여백, 폰트, 포인트 컬러가 가독성의 핵심** — 정보 밀도보다 읽히는 속도.
- **GitHub/포트폴리오 링크 첨부가 사실상 필수** (2025~2026 개발자 채용 트렌드).
- **현대차 1페이지 자소서는 올해 처음 도입** — 레퍼런스가 없으므로 튀는 게 유리.
  단, 콘텐츠 없는 디자인은 역효과.

---

## 1. 선택지 비교

| 항목 | 일반 워드 | 디자인 PDF (세로형) |
|---|---|---|
| 가독성 (모바일) | ✅ 안정적 | ✅ 안정적 (세로형 선택 시) |
| 차별화 | ❌ 수백 장 중 묻힘 | ✅ 시각적으로 기억에 남음 |
| 제작 난이도 | 낮음 | 중간 (내가 생성) |
| 인상 | 무난함 | "이 사람은 디자인 감각도 있다" |
| 위험 | 내용만으로 승부 | 디자인이 내용을 압도하면 역효과 |

**결론: 디자인 PDF (세로형 단단 레이아웃) 선택**

---

## 2. 레이아웃 설계

### 기본 원칙
- **단단(단일 컬럼)** — 모바일 가독성 보장
- **섹션 구분선** 으로 시각 흐름 제어 (2단 없이도 구조감 생성)
- **포인트 컬러 1개** — 현대차 브랜드 컬러 계열 (딥 네이비 `#1F3864` 또는 `#0047AB`)
- **여백 상하좌우 2cm 이상** — 여백이 넓을수록 고급스러워 보임

### 구조 (위→아래)
```
┌─────────────────────────────────┐
│  [헤더 블록]                     │
│  이름 (크고 굵게)                 │
│  숭실대학교 컴퓨터공학 | 금융학부 부전공  │
│  github.com/xxx  ·  email       │
├─────────────────────────────────┤
│  [기술 스택 한 줄]                │
│  Python · FastAPI · LangGraph · │
│  Neo4j · Qdrant · Prometheus    │
├─────────────────────────────────┤
│  [오프닝 — 1~2문장]               │
│  행위로 시작. 이 사람이 누구인지.  │
├─────────────────────────────────┤
│  [본론 1 — 핵심 기술 경험]         │
│  프로젝트 + JD 매핑 기술 요소      │
├─────────────────────────────────┤
│  [본론 2 — 문제 해결 사례]         │
│  상황 → 선택 → 결과               │
├─────────────────────────────────┤
│  [마무리 — 직무 연결 + 차별점]      │
│  CS + 금융 한 문장                │
│  사고방식 한 문장으로 마무리        │
└─────────────────────────────────┘
```

## 3. 타이포그래피 & 색상

| 요소 | 사양 |
|---|---|
| 폰트 | 맑은 고딕 (본문) / Pretendard or 나눔고딕ExtraBold (이름) |
| 본문 크기 | 10~10.5pt |
| 소제목 | 11~12pt, 볼드 |
| 이름 | 18~20pt, 볼드 |
| 포인트 컬러 | 딥 네이비 `#1F3864` (헤더, 섹션 라인) |
| 본문 색상 | 다크 그레이 `#2D2D2D` (순수 검정보다 눈에 부드러움) |
| 배경 | 흰색 |
| 강조 | 볼드 또는 포인트 컬러 — 한 섹션에 1~2개만 |

## 4. 피해야 할 것

- ❌ 가로 2단 레이아웃 — 모바일 가독성 저하
- ❌ 컬러 3개 이상 사용 — 산만해 보임
- ❌ 사진 삽입 — 자소서에는 불필요, 공간 낭비
- ❌ 테이블/그리드 과다 — 이력서가 아니라 자소서임
- ❌ 배경 색상 사용 — 프린트 시 잉크 과다 소비, 깔끔함 저해
- ❌ 아이콘 남발 — 1~2개 이내 (섹션 구분 목적만)

## 5. GitHub README 준비 사항 (자소서 제출 전 완료)

자소서에 링크를 넣으려면 GitHub가 먼저 정비되어 있어야 한다.

- [ ] README 상단: 프로젝트 한 줄 요약
- [ ] 기술 스택 배지 (shields.io)
- [ ] 아키텍처 다이어그램 또는 플로우차트 (이미지)
- [ ] 주요 기능 설명 (JD 키워드 자연스럽게 포함)
- [ ] 실행 방법 (간단하게)
- [ ] 코드 최소 정리 (민감 키 .env로 분리, .gitignore 확인)

## 6. 제작 방식

Python(WeasyPrint 또는 ReportLab)으로 HTML/CSS → PDF 자동 생성.
내가 직접 제작. 지성 씨는 내용만 확정하면 됨.

### 작업 순서
1. 자소서 내용 초안 확정 (텍스트)
2. GitHub README 정비
3. 양식 시안 제작 (PDF)
4. 피드백 후 최종 PDF 출력
5. 제출
